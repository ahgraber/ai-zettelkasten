| ------------------------------- | ----------------------- | ---------------------------------------------------------- |
| Source identity table           | `bookmarks`             | `sources`                                                  |
| `karakeep_id` nullability       | NOT NULL                | NULL (optional)                                            |

## Table: `sources`

Replaces the old `bookmarks` table.
Stores the canonical durable identity for anything the system can convert.
The `karakeep_id` column is now nullable to allow future source kinds (arXiv IDs, raw URLs, inline HTML, GitHub repos) that are not backed by a KaraKeep bookmark.

| Column            | Type         | Nullable | Notes                                            |
| ----------------- | ------------ | -------- | ------------------------------------------------ |
| `id`              | INTEGER      | NOT NULL | Primary key                                      |
| `karakeep_id`     | VARCHAR(255) | NULL     | Unique; NULL for non-KaraKeep sources            |
| `aizk_uuid`       | CHAR(32)     | NOT NULL | Unique; FK target for child tables               |
| `source_ref`      | TEXT         | NULL     | Canonical reference string, e.g. `karakeep:<id>` |
| `source_ref_hash` | TEXT         | NULL     | SHA-256 of `source_ref`; unique index            |
| `url`             | VARCHAR      | NULL     |                                                  |
| `normalized_url`  | VARCHAR      | NULL     | Indexed                                          |
| `title`           | VARCHAR(500) | NULL     |                                                  |
| `content_type`    | VARCHAR(10)  | NULL     |                                                  |
| `source_type`     | VARCHAR(20)  | NULL     |                                                  |
| `created_at`      | DATETIME     | NOT NULL |                                                  |
| `updated_at`      | DATETIME     | NOT NULL |                                                  |

Indexes: `ix_sources_aizk_uuid` (unique), `ix_sources_karakeep_id` (unique),
`ix_sources_normalized_url`, `ix_sources_source_ref_hash` (unique).

## Table: `conversion_jobs`

New column: `source_ref TEXT NULL` — denormalized copy of `sources.source_ref` for the job's source.
FK retargeted from `bookmarks.aizk_uuid` → `sources.aizk_uuid`.

`idempotency_key` is recomputed during migration using `sha256(f"{source_ref_hash}:docling:{old_key}")`.

## Table: `conversion_outputs`

No structural column changes.
FK retargeted from `bookmarks.aizk_uuid` → `sources.aizk_uuid`.
